O desafio: construir uma base sólida sem investimento pesado

Quando a Scindax nasceu, tínhamos um problema claro: precisávamos de uma presença digital profissional, um sistema para captar leads e uma base de dados segura para gerenciar tudo isso.

O orçamento? Praticamente zero.

A solução? Usar o que há de melhor no mercado sem precisar pagar por licenças caras ou contratar uma equipe de infraestrutura desde o primeiro dia.

Foi assim que criamos um ecossistema digital completo usando ferramentas que escalam com o crescimento, mas que começam totalmente gratuitas.

A arquitetura que criamos

1. O site institucional (Cloudflare Pages)

Precisávamos de um site que passasse credibilidade e profissionalismo, mas que não exigisse manutenção constante nem custos de hospedagem.

Optamos pelo Cloudflare Pages por três motivos:

  • Gratuito para começar.
  • Integração automática com o GitHub — cada atualização no código publica o site em segundos.
  • CDN global e SSL automático — o site carrega rápido em qualquer lugar do mundo, com segurança de ponta.

Na prática: criamos um repositório no GitHub, conectamos ao Cloudflare Pages, e em menos de 30 minutos tínhamos o site da Scindax no ar, com HTTPS ativo e performance excelente. Hoje, qualquer atualização que fazemos no código já está disponível para o público automaticamente.

2. O formulário de captura (Tally)

A Avaliação de Maturidade Empresarial (AMES) é a porta de entrada da Scindax. Precisávamos de um formulário que fosse:

  • Fácil de criar e manter.
  • Integrável com nossa infraestrutura.
  • Capaz de coletar dados estruturados.

Usamos o Tally para isso. É uma ferramenta que nos permite criar formulários com uma experiência muito próxima de um produto SaaS, mas sem custos de desenvolvimento inicial.

O diferencial: o Tally permite webhooks — ou seja, cada resposta enviada pelo usuário pode ser automaticamente enviada para um sistema que construímos.

3. A API e o banco de dados (Cloudflare Workers + D1)

Este foi o coração da nossa arquitetura.

Precisávamos de um lugar seguro para armazenar os dados da AMES, que pudesse ser consultado pelo nosso sistema de gestão interno, mas sem expor informações sensíveis ao público.

Criamos um Worker no Cloudflare, que é basicamente um código que roda em servidores da Cloudflare e funciona como nossa API particular.

E para armazenar os dados, usamos o Cloudflare D1 — um banco de dados SQL que é:

  • Rápido.
  • Escalável.
  • E, novamente, com custos muito baixos para começar.

O fluxo ficou assim:

Usuário preenche o formulário no Tally
         ↓
O Tally envia os dados para o Worker via webhook
         ↓
O Worker valida e grava no banco D1
         ↓
Nosso sistema de gestão consulta o banco via API
         ↓
Analisamos as respostas e geramos insights

Resultado: os dados dos nossos leads nunca ficam expostos publicamente. Tudo é armazenado em um banco de dados que só a Scindax pode acessar.

4. O sistema de gestão (nosso "Manager")

Toda essa infraestrutura não serviria de nada se não pudéssemos enxergar e analisar os dados que estávamos coletando.

Desenvolvemos um sistema de gestão interno (chamamos carinhosamente de "Manager") que consome a API criada no Worker e exibe as avaliações em uma interface amigável.

Esse sistema permite que a equipe da Scindax:

  • Visualize todas as respostas da AMES.
  • Acompanhe o fluxo de leads.
  • Identifique padrões e oportunidades.
  • Gerencie o relacionamento com os clientes.

E o melhor: o Manager se conecta diretamente à nossa API, sem depender de arquivos ou planilhas intermediárias. Tudo em tempo real, seguro e organizado.

Quanto custou tudo isso?

Essa é a parte que mais surpreende.

A infraestrutura inteira — site, API, banco de dados, integração — custou menos de R$ 50 por mês para começar.

E o melhor: enquanto o volume de leads é baixo, a maior parte dos recursos permanece no plano gratuito das plataformas que usamos.

Quando a Scindax crescer e tiver milhares de avaliações por mês, a arquitetura já está preparada para escalar — e aí sim, pagaremos proporcionalmente ao uso, sem surpresas.

O que aprendemos com essa jornada

1. É possível construir infraestrutura profissional sem investimento inicial alto.
A combinação de Cloudflare, Tally e um sistema de gestão próprio nos deu uma base sólida por menos do que o custo de uma hospedagem tradicional.

2. Dados não podem ficar expostos.
No MVP usamos Google Sheets e, mesmo com todas as proteções, sabíamos que não era ideal. Migrar para um banco de dados próprio foi uma decisão estratégica desde o início.

3. Automatize tudo o que for possível.
Desde o envio do formulário até a consulta no sistema de gestão, tudo é automático. Isso reduz erros, economiza tempo e nos permite focar no que realmente importa: ajudar empresas a crescerem.

4. Escolha ferramentas que cresçam com você.
Cloudflare Workers e D1 são plataformas que começam gratuitas, mas permitem escalar para milhões de requisições sem precisar migrar de infraestrutura. É o tipo de decisão que evita dores de cabeça no futuro.
